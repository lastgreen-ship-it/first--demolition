# Claude Code에 그대로 붙여넣는 최종 실행 프롬프트

당신은 퍼스트철거의 시니어 풀스택 개발 리드다. 이 저장소의 `CLAUDE.md`, `docs/개발팀_최종작업지시서.md`, `docs/reference/퍼스트철거_사업계획서_최종안.pdf`를 먼저 읽고 작업하라.

목표는 현재 시연용 MVP를 실제 운영 가능한 P0 버전으로 완성하는 것이다.

작업 순서:
1. 전체 코드베이스를 점검하고 구현 현황/누락/보안위험을 `docs/AUDIT.md`에 기록한다.
2. Supabase schema를 migration 방식으로 정리하고 profiles, quote_requests, quote_photos, vendors, bids, projects, project_evidence, settlements의 RLS를 역할별로 완성한다.
3. 고객 인증, 견적 접수, 사진 5~10장 private upload, 접수완료 화면을 구현한다.
4. 관리자 로그인, 견적 목록/검색/필터/상세/상태변경/담당자 메모를 구현한다.
5. 협력업체 로그인, 검증서류 등록, 배정 건 열람, 입찰 제출을 구현한다.
6. 고객 입찰 비교 및 업체 선정, 프로젝트 타임라인, 시공 전/중/후와 폐기물 증빙을 구현한다.
7. 모든 비밀키는 서버 전용으로 유지하고 파일은 signed URL로만 조회한다.
8. 모바일 320px부터 데스크톱까지 반응형과 접근성을 검수한다.
9. Vitest 또는 Playwright 기반 핵심 테스트를 추가한다.
10. `npm run typecheck`, `npm run lint`, `npm run build`가 모두 성공할 때까지 수정한다.
11. 완료 후 `docs/DEPLOYMENT.md`, `docs/OPERATIONS.md`, `docs/TEST-RESULTS.md`를 작성한다.

제약:
- 사업계획서에 없는 기능을 임의로 크게 확장하지 않는다.
- 예상견적은 확정금액처럼 표시하지 않는다.
- 실제 외부 API가 정해지지 않은 기능은 adapter interface와 mock으로 분리한다.
- 데이터 삭제나 schema 파괴가 필요한 경우 먼저 이유와 안전한 migration 계획을 제시한다.

먼저 감사보고서를 작성하고, P0 구현 계획을 체크리스트로 제시한 뒤 바로 코딩을 시작하라.
