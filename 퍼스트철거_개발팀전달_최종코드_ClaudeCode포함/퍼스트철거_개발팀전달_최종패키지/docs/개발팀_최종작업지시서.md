# 퍼스트철거 개발팀 최종 작업지시서

## 1. 목표
철거 고객의 견적 접수부터 검증업체 경쟁입찰, 업체선정, 시공관리, 현장증빙, 완료검수, 정산까지 처리하는 운영형 MVP를 구축한다.

## 2. 사용자 역할
- 고객: 견적 접수, 사진 등록, 입찰 비교, 업체선정, 진행확인, 완료확인
- 협력업체: 서류제출, 검증상태 확인, 배정 견적 열람, 입찰, 시공증빙
- 자체시공팀: 배정현장 일정, 작업보고, 사진·폐기물 증빙
- 관리자: 전체 고객/업체/견적/입찰/프로젝트/정산/KPI 관리

## 3. P0 필수 기능
- 반응형 랜딩 및 견적 신청
- 필수 정보: 이름, 연락처, 주소, 공사유형, 면적, 희망일, 요청사항
- 사진 최소 5장/최대 10장, 용량·확장자 검사, 비공개 저장
- 접수번호 자동발급, 상태 워크플로
- 관리자 목록/검색/필터/상세/메모/담당자
- 개인정보 동의와 예상견적 면책 고지
- Supabase Auth, DB, Storage, RLS

## 4. P1 기능
- 업체 검증: 사업자등록증, 보험, 실적, 서비스지역
- 입찰: 금액, 기간, 포함/제외, 유효기간
- 고객 비교표와 업체 선택
- 프로젝트 단계: 계약대기→착공→진행→완료검수→정산완료
- 시공 전/중/후, 폐기물, 문서 증빙

## 5. AI 예상견적
초기에는 관리자 가격표 기반 규칙식으로 구현하고, 데이터가 축적되면 회귀/추천모델로 전환한다. 결과는 단일 금액이 아니라 하한~상한 범위와 근거항목을 표시한다.

## 6. 보안
- Service Role Key 브라우저 노출 금지
- 모든 테이블 RLS 활성화 및 역할별 테스트
- 파일 private bucket + signed URL
- 전화번호 마스킹, 관리자 접근로그
- 입력 검증, 속도 제한, 봇 방지

## 7. 배포/도메인
- preview: Vercel 자동 배포
- demo: `demo.firstdemolition.co.kr`
- production: `firstdemolition.co.kr`
- dev/staging/prod Supabase 분리 권장

## 8. 인수 테스트
- iPhone/Android/Chrome/Edge 320~1440px
- Lighthouse 기본 성능/접근성 점검
- 권한별 비인가 접근 차단
- 5~10장 업로드, 실패 재시도
- 견적→입찰→선정→완료→정산 E2E
- 백업/복구와 관리자 운영 매뉴얼

## 9. 납품물
