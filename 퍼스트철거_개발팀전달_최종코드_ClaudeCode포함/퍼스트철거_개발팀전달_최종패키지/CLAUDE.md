# Claude Code 실행 규칙 - 퍼스트철거

## 목적
사업계획서와 개발 작업지시서에 맞춰 퍼스트철거 운영형 MVP를 구현한다. 시연용 화면에서 끝내지 말고 Supabase 데이터, 인증, RLS, Storage, 관리자 운영흐름까지 검증한다.

## 필수 원칙
1. 사용자 화면은 한국어, 모바일 우선, 320px부터 정상 작동.
2. 고객/협력업체/자체시공팀/관리자 권한을 분리.
3. 개인정보와 사진 버킷은 비공개. Service Role Key는 서버에서만 사용.
4. 가격은 '예상 범위'로 표시하고 확정 견적으로 오인되지 않게 고지.
5. 위험 작업, 석면, 구조물, 폐기물은 별도 확인 체크를 둔다.
6. 모든 변경 후 `npm run typecheck`, `npm run lint`, `npm run build` 수행.
7. 데이터베이스 변경은 반드시 새 SQL migration으로 남긴다.
8. 임의의 가짜 외부 API 키나 개인정보를 커밋하지 않는다.

## 구현 우선순위
P0: 공개 랜딩, 견적폼, 사진 5~10장, 관리자 접수목록, 상태 변경, RLS
P1: 업체 가입/검증, 입찰, 업체선정, 프로젝트 타임라인, 현장증빙
P2: 계약/결제/정산, 알림, 분석 대시보드, AI 가격추천

## 완료 정의
- 신규 사용자가 모바일에서 3분 이내 견적 접수
- 관리자가 접수 확인 후 상태 변경
- 업체는 배정된 건만 열람하고 입찰
- 고객은 본인 건만 열람
- 비공개 파일은 signed URL로만 조회
- 운영 로그와 에러 처리가 존재
